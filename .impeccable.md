# Project Design Context — AION-MEDSAFE

This file guides all design work (the `impeccable` skill reads it). The site is a
**GitHub Pages landing page** for the open-source project and the Sealed Evidence
Packet (SEP) standard.

## The one idea (write everything to serve this)

**"Turn investigative leads into court-defensible, independently verifiable
evidence — in the open."** Not another black-box risk score; a transparent,
tamper-evident chain of custody anyone can check.

## Message hierarchy

1. **Remember:** AION-MEDSAFE makes Medicaid provider-integrity evidence
   *verifiable* (sealed, hash-chained, signed) — and the **SEP** format is an open
   standard for it.
2. **Proof:** real numbers (7 public sources incl. 9.55M NPPES records, 8
   detection signals), the four verification guarantees, open code + spec, and
   plainly stated limitations.
3. **Action (CTAs, in priority):** Read the SEP spec → Understand how it works →
   Request a pilot. (GitHub link always present in nav.)

## Design Context

### Users
- **Primary: OSS adopters & the technically curious** (integrity-tech developers,
  contributors), with credibility content that also serves **government buyers**
  (Med-QUEST Program Integrity, MFCU, state IT/CISO) and the **standards/policy**
  community. Read on a workstation, often skeptical, scanning for substance.
- Job to be done: quickly judge "is this real, is it trustworthy, and what do I do
  next" — then read the spec, grasp the provenance model, or start a pilot.

### Brand Personality
- Three words: **precise, candid, institutional.**
- Voice: plain, evidence-led, unhurried, never hype. Proof over adjectives.
  Transparency (including limitations) is a credibility asset, not a weakness.
- Emotional goal: earned **trust** and quiet authority — the feeling of a serious,
  official record, not a startup pitch.

### Aesthetic Direction
- **"Official record / sealed document / forensic ledger."** Institutional
  editorial: ink-on-paper restraint, precise typography, generous space, evidence
  laid out the way a sealed dossier would be. Provenance (seals, hashes,
  verification) is the recurring visual motif — used meaningfully, never as
  decoration.
- **Theme: light, institutional.** Warm-paper neutrals (tinted, never pure white),
  near-black ink text (tinted, never pure black). One confident **signal color**
  used sparingly (~10% by weight) for the "seal / verify / action" moments — derive
  a distinctive, non-cliché hue in OKLCH (a wax-seal / official-stamp register is
  on-theme and avoids the trap below). Tint neutrals toward that hue.
- **Anti-references (hard no):** the AI "security product" cliché — dark mode with
  neon cyan/green glow, purple-to-blue gradients, gradient text, glassmorphism,
  side-stripe accent borders, icon-topped card grids, big hero metric template.
  No "mono = technical" lazy shorthand. It must not read as AI-generated.
- Typography: pair a distinctive display face with a refined body face (run the
  font-selection procedure; avoid the reflex/banned list). A monospace is allowed
  ONLY for actual hashes/code/verification snippets — as evidence, not vibe.

### Design Principles
1. **Proof over adjectives.** Show hashes, figures, the four guarantees, real
   source counts — not "powerful/secure/trusted."
2. **Restraint is credibility.** Institutional calm, one signal color, lots of
   space. Hype reads as overcompensation to this audience.
3. **Evidence as aesthetic.** Treat the page like a sealed record; make
   verifiability visible and, where possible, demonstrable.
4. **Clarity first.** One idea, clear message hierarchy, headlines that land in one
   read; plain language over jargon.
5. **No AI slop, no cyber clichés.** Honor the skill's bans; distinctive, authored,
   light-institutional — someone should ask "how was this made?", not "which AI?"

## Build/deploy constraints
- **GitHub Pages**, static, no build step (vanilla HTML/CSS/JS preferred).
- Output to **`docs/site/`** (the repo's `docs/` already holds Markdown), include
  `.nojekyll`, use **relative paths** (project site serves under `/aion-medsafe/`).
- Ship `<head>` essentials + a deliberately designed Open Graph share image.
- Accessible (semantic landmarks, focus states, contrast), reduced-motion aware,
  Lighthouse ≥ 95, no layout shift.
- Carry the repo's **affiliation/disclaimer** (independent prototype, not
  affiliated/endorsed, public data only, leads-not-findings).
